#录波文件传输
在属性fileInfo中，使用totalFrames和curFrameIdx配合完成需要图片文件的传输，curFrameIdx从1开始依次递增，传输最后一个数据帧为0，表示文件传输完成。如果数据只有1帧，这curFrameIdx为0进行传输。
fileContent内容为文件数据转化而成的十六进制的字符串。
fileFormat为传输文件的格式，可以为空，curFrameIdx为0时必须存在该字段，方便接受方定义整个文件的格式
fileMD5为整个文件的MD5校验码转化而成的十六进制的字符串，curFrameIdx为0时必须存在该字段，用于整个文件的一致性校验。
fileName为传输文件的文件名,curFrameIdx为0时存在这个字段。
示例：
{
	"devices": [{
		"deviceId": "XXXXXXXXXXXXXXXX",
		"services": [{
			"data": {
				"fileInfo": {
				"totalFrames": 3,
				"curFrameIdx": 1,
				"fileContent": "XXXXXXXXXXXXXXXXXXXXXXXXXXXX",
				"fileFormat": "cfg/dat",
				"fileMD5": "XXXXXXXXXXXXXXXX",
				"fileName": "BAY01_0058_20201103_213142_173.cfg"
				}
			},
			"eventTime": "20220316T092113Z",
			"serviceId": "fileTransfer"
		}]
	}]
}
